## Metodologia escolhida (Scrum)

A metodologia Scrum é uma estrutura ágil para o desenvolvimento de produtos complexos, ideal para projetos que exigem flexibilidade, colaboração e entregas incrementais. Para o desenvolvimento de um compilador de C++ para javascrpt, o Scrum oferece uma abordagem robusta e adaptável, justificando sua escolha por diversas razões-chave.

## Projeto complexo e incremental

Um compilador, mesmo reduzido, é um software de alta complexidade com varias etapas interdependentes.O Scrum permite dividir esse grande problema em partes menores e entregáveis, cada uma agregando valor tangível,Isso torna o desenvolvimento mais controlável, testável e ajustável a cada ciclo.

## Colaboração constante e revisão contínua

A metodologia incentiva reuniões curtas e frequentes (daily scrums) e demonstrações de progresso (sprint reviews).
Isso é perfeito para um projeto técnico, pois:

- permite feedback rápido sobre cada módulo (por exemplo, testes de tradução de funções ou estruturas de controle),

- ajuda a alinhar o time sobre decisões técnicas e padrões de código,

- reduz o risco de falhas acumuladas.

  ## Entregas funcionais e testáveis a cada sprint

  Essas entregas permitem testes reais e demonstrações progressivas da evolução do compilador.
